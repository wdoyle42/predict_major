# predict_major

**Predicting College Major Field of Study: A Neural Network Approach Using the Education Longitudinal Study of 2002**

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

## Overview

This repository is a fully reproducible research compendium for a study
predicting bachelor's degree major field (2-digit CIP code) from student
background characteristics, psychosocial attitudes, high school coursetaking
trajectories, and early postsecondary experiences, using data from the
[Education Longitudinal Study of 2002 (ELS:2002)](https://nces.ed.gov/surveys/els2002/).

The analysis uses a feedforward neural network implemented in the
[tidymodels](https://www.tidymodels.org/) framework with a
[keras](https://keras.rstudio.com/) engine, managed via a
[targets](https://docs.ropensci.org/targets/) pipeline.

## Data Access

The ELS:2002 restricted-use data file
(`els_02_12_byf3pststu_v1_0.dta`) is **not included** in this repository.
It is available from the National Center for Education Statistics under a
restricted-use data license:

- Apply for access: https://nces.ed.gov/statprog/rudman/
- ELS:2002 study page: https://nces.ed.gov/surveys/els2002/

Once you have the file, place it at:

```
data/raw/els_02_12_byf3pststu_v1_0.dta
```

## Reproducing the Analysis

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/predict_major.git
cd predict_major
```

### 2. Restore the R package environment

```r
# In R
install.packages("renv")
renv::restore()
```

### 3. Install keras / TensorFlow backend

```r
library(keras)
install_keras()   # installs a Python virtualenv with TensorFlow
```

### 4. Place the data file

See **Data Access** above.

### 5. Run the pipeline

```r
library(targets)
tar_make()
```

To visualize the pipeline dependency graph:

```r
tar_visnetwork()
```

### 6. Render the paper

The paper renders automatically as the final step of `tar_make()`. To render
manually after the pipeline has run:

```r
targets::tar_render(paper, "paper/paper.Rmd")
targets::tar_render(appendix, "paper/appendix.Rmd")
```

## Pipeline Overview

```
raw data → analytic sample → recipe → split/folds →
model spec → workflow → tuning → final fit →
metrics → figures → tables → paper
```

Each stage is a named target in `_targets.R`. Targets caches all
intermediate objects; only stale targets are recomputed on subsequent
`tar_make()` calls.

## Notes on Reproducibility

- All random seeds are set to `2002` (ELS base year) throughout.
- Package versions are locked via `renv.lock`.
- The `_targets/` cache is not committed to git; the pipeline must be run
  locally to reproduce all objects.
- The `data/raw/` and `data/processed/` directories are gitignored due to
  NCES data use restrictions.
